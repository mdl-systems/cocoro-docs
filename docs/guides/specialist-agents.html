👔 専門職エージェントの使い方

Cocoro OS のエージェントシステムは、6 種の専門職 が協力して複雑なタスクを自律実行します。 「調べて」「まとめて」「スケジュールに入れて」といった複合指示も、AI が自動でパイプライン実行します。

6 種の専門職エージェント​

#エージェント役割主なツール
1🔍 リサーチャーWeb 検索・情報収集・まとめweb_search, search_learnings
2📅 スケジューラー予定管理・リマインダーadd_schedule, list_schedules
3✅ タスクマネージャーToDo 追跡・優先度管理create_task, list_recent_tasks
4🧠 メモリキュレーター記憶の整理・引用・学習search_memory, search_learnings
5📊 アナリストデータ分析・要約・構造化全ツール + LLM 集中推論
6🛠️ カスタムエージェントユーザー定義の専門タスクget_org_status + カスタム

エージェントの選択ロジック​

Cocoro OS はメッセージの内容に応じて自動的にエージェントを選択します：

入力: 「明日の14時に打ち合わせを入れて、関連資料を調べてまとめて」

Cocoro の内部処理:
  1. タスク分解
     - タスクA: スケジュール登録 → スケジューラー担当
     - タスクB: 資料リサーチ → リサーチャー担当

  2. 並列実行
     [スケジューラー] add_schedule("打ち合わせ", "2026-03-15 14:00")
     [リサーチャー]   web_search("打ち合わせ 関連資料")
                     search_memory("打ち合わせ 過去の議事録")

  3. 結果統合 → 返答生成

🔍 リサーチャーエージェント​

基本的な使い方​

ユーザー: 「2026年の生成AIトレンドトップ3を調べてまとめて」

Cocoro:  [web_search] "2026 生成AI トレンド" を検索中...
         [search_memory] 過去の関連記憶を検索中...
         [search_learnings] 学習済み知識を検索中...

         📊 2026年の生成AIトレンド トップ3

         1. **エージェント型AIの台頭**
            自律的に計画・実行するAIが標準化。Cocoro OSのような
            パーソナルAIエージェントが一般消費者向けに普及。

         2. **ローカルLLMの実用化加速**
            Ollama + Gemma2 などの組み合わせで、クラウド不要の
            高精度推論が可能になりつつある。

         3. **マルチモーダルの標準化**
            音声・画像・テキストを統合したマルチモーダルが
            APIの標準機能として提供される。

         Sources: TechCrunch, MIT Technology Review, The Verge

複合リサーチ（キラー機能）​

「先月話したプロジェクトの件で、競合他社の動向を調べて比較表を作って」

Cocoro:  [search_memory] "プロジェクト" で過去の会話を自動検索...
         → 3件の関連記憶を発見（先月14日・先月28日の会話）

         [web_search] "競合他社名 最新動向 2026"...

         前回の会話内容を踏まえた上で比較表を生成:
         （先月の会話を「記憶」として活用）

📅 スケジューラーエージェント​

ユーザー: 「明日の午後2時に田中さんとのMTGを1時間入れて。
           場所はZoom。前日にリマインドをお願い」

Cocoro:  [add_schedule] {
           "title": "田中さんとのMTG",
           "date": "2026-03-15",
           "time": "14:00",
           "duration": 60,
           "location": "Zoom",
           "reminder": "2026-03-14T14:00"
         }

         ✅ 登録しました！
         📅 2026年3月15日（土）14:00〜15:00
         📍 Zoom
         ⏰ 前日 14:00 にリマインドします

スケジュール確認​

ユーザー: 「今週の予定を教えて」

Cocoro:  [list_schedules] week=current

         今週のスケジュール（2026/3/11〜3/17）:

         月 3/11:  10:00 週次レビュー（1h）
         火 3/12:  なし
         水 3/13:  15:00 設計レビュー（2h）
         木 3/14:  なし
         金 3/15:  14:00 田中さんとのMTG（1h）
         土・日:   予定なし

         📊 今週の予定: 3件（合計 4 時間）

✅ タスクマネージャーエージェント​

ユーザー: 「今週の未完了タスクを優先度順に教えて」

Cocoro:  [list_recent_tasks] status=pending, week=current, sort=priority

         今週の未完了タスク（4件）:

         🔴 高優先: cocoro-console テスト実装
                   期限: 3/15（明日）
                   進捗: 60% 完了

         🔴 高優先: デプロイ手順書の更新
                   期限: 3/15
                   進捗: 未着手

         🟡 中優先: README 英語翻訳
                   期限: 3/17
                   進捗: 未着手

         🟢 低優先: ブログ記事の草稿
                   期限: 未設定
                   進捗: 30% 完了

タスクの作成​

ユーザー: 「来週末までに API ドキュメントを書くタスクを高優先で登録して」

Cocoro:  [create_task] {
           "title": "API ドキュメント作成",
           "priority": "high",
           "deadline": "2026-03-21",
           "estimated_hours": 3
         }

         ✅ タスクを登録しました。
         「API ドキュメント作成」（高優先 / 期限: 3/21）

📊 アナリストエージェント​

長文の要約・データ分析・構造化が得意です。

ユーザー: 「この会議メモを要約して、アクションアイテムを抽出して」

ユーザー: （長い会議メモをペースト）

Cocoro:  📝 会議要約
         日時: 2026/3/14、参加者: 5名

         主要な議論:
         - 来期ロードマップの優先順位
         - Q2 のリソース配分
         - 外部依存ライブラリの更新戦略

         ✅ アクションアイテム（3件）:
         1. 田中さん → ロードマップ v2 ドラフト作成（期限: 3/20）
         2. 鈴木さん → リソース計画書の提出（期限: 3/18）
         3. 全員 → 次回 MTG 出席確認（期限: 3/15）

         タスクに登録しますか？ [はい / いいえ]

API でタスクを直接実行する​

# タスクを投げる
curl -X POST http://cocoro.local:8001/agent/task \
  -H "Authorization: Bearer $COCORO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "競合サービスの料金プランを調査してまとめる",
    "type": "research",
    "priority": "high",
    "deadline": "2026-03-14T18:00:00Z"
  }'

# 応答例
{
  "task_id": "task_a1b2c3",
  "status": "running",
  "created_at": "2026-03-14T10:30:00Z"
}

# タスク状態を確認
curl http://cocoro.local:8001/agent/task/task_a1b2c3 \
  -H "Authorization: Bearer $COCORO_API_KEY"

エージェントのパフォーマンスを上げるコツ​

1. 具体的に指示する​

❌ 「何か調べて」
✅ 「Cloudflare Tunnel の設定方法を調べて、手順を箇条書きで5ステップにまとめて」

❌ 「タスクを整理して」
✅ 「今週期限のタスクを優先度順に並べて、今日やるべき3つを教えて」

2. 複合タスクを一括指示する​

「今週のAIニュースを調べて、要点をまとめて、タスクに"週次AIウォッチ"として登録して」

→ web_search → analyze → create_task を自動でパイプライン実行

3. 過去の会話を引用させる​

「先月の◯◯プロジェクトの件を踏まえて、今月の進め方を提案して」

→ search_memory で自動的に関連する過去の会話を引用
