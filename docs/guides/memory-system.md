# 🧠 記憶・学習システム

Cocoro OS の最大の差別化機能が**3層記憶システム**です。
会話を重ねるごとに記憶が蓄積され、次の会話でその記憶を自動的に活用します。

---

## 3層記憶アーキテクチャ

```
┌─────────────────────────────────────────────────────────┐
│                   記憶システム全体像                       │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Layer 1: 短期記憶（Short-Term Memory）          │   │
│  │  技術: Redis                                     │   │
│  │  TTL: 24時間                                     │   │
│  │  内容: 現在の会話スレッド（最新 20 ターン）         │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓ 自動アーカイブ（24h後）        │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Layer 2: 長期記憶（Long-Term Memory）           │   │
│  │  技術: PostgreSQL                                │   │
│  │  TTL: 永続（重要度スコアで自動整理）               │   │
│  │  内容: 重要な出来事・学習・ユーザー情報             │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓ ベクトル化                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Layer 3: ベクトル記憶（Vector Memory）          │   │
│  │  技術: pgvector（PostgreSQL 拡張）               │   │
│  │  内容: 全記憶のセマンティック検索インデックス        │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

---

## 記憶の種類

| 種別 | 説明 | 例 |
|------|------|----|
| **会話記憶** | 実際の会話ターン | 「田中さんの件を相談した」 |
| **ユーザー情報** | あなたについての情報 | 「エンジニア、料理が趣味」 |
| **学習事項** | 新たに学んだ知識 | 「Cocoro OSのアーキテクチャ説明」 |
| **感情的な出来事** | 重要な感情的記憶 | 「嬉しかった・悲しかった出来事」 |
| **タスク記録** | 完了したタスクの記録 | 「〇〇プロジェクト完了」 |

---

## 記憶の自動保存フロー

会話中に自動的に記憶が作られます：

```
会話ターン
    │
    ▼
┌─────────────────────────────────────────────────┐
│  重要度スコアリング（0〜1.0）                      │
│  - 感情的インパクト: 高い感情 → スコア上昇         │
│  - 新規性: 初めて聞く情報 → スコア上昇             │
│  - ユーザー情報: 個人情報 → 自動高スコア           │
│  - 繰り返し度: よく言及される → スコア上昇         │
└─────────────────────────────────────────────────┘
    │
    ▼
スコア > 0.3 → 長期記憶に保存
スコア ≤ 0.3 → 24時間後に自動削除（短期記憶のみ）
```

## 記憶の API

### 記憶一覧の取得

```bash
GET /memory/list?limit=20&offset=0
```

```json
{
  "memories": [
    {
      "id": "mem_abc123",
      "content": "cocoro-docs のドキュメント拡充作業について話した",
      "importance": 0.85,
      "created_at": "2026-03-14T10:30:00Z",
      "memory_type": "conversation",
      "tags": ["work", "cocoro-docs"]
    }
  ],
  "total": 1482,
  "page": 1
}
```

### セマンティック検索

```bash
POST /memory/search
Content-Type: application/json

{
  "query": "プロジェクトの進捗",
  "limit": 5,
  "threshold": 0.7
}
```

```json
{
  "results": [
    {
      "id": "mem_xyz789",
      "content": "cocoro-docs の15ページ追加が完了した",
      "similarity": 0.92,
      "created_at": "2026-03-13T15:00:00Z"
    }
  ]
}
```

### 手動で記憶を追加

```bash
POST /memory/add
Content-Type: application/json

{
  "content": "会社の主力製品はAIエージェントシステム",
  "importance": 0.9,
  "memory_type": "user_info",
  "tags": ["work", "context"]
}
```

### 記憶の削除

```bash
DELETE /memory/mem_abc123
```

## 学習システム（Learnings）

会話から自動的に「学習事項」を抽出・保存します：

```bash
GET /memory/learnings?topic=AI&limit=10
```

```json
{
  "learnings": [
    {
      "id": "learn_001",
      "topic": "Cocoro OS",
      "content": "シンクロ率が 92% を超えると Divergence Ceiling が発動し学習停止",
      "confidence": 0.95,
      "source": "会話 2026-03-08"
    }
  ]
}
```

## 記憶の重要度を理解する

| スコア | 意味 | 保持期間 |
|--------|------|---------|
| 0.9〜1.0 | 非常に重要（感情的・人格的な核心）| 永続 |
| 0.7〜0.9 | 重要（よく参照される情報）| 永続 |
| 0.5〜0.7 | 普通（有用な情報）| 6ヶ月〜永続 |
| 0.3〜0.5 | やや重要 | 30日 |
| 0.0〜0.3 | 一般的な会話 | 24時間（短期のみ）|

## 記憶システムのメンテナンス

### アーカイバーの手動実行

古い記憶の整理・圧縮を手動で実行：

```bash
# アーカイブ実行（重要度低い古い記憶を圧縮）
curl -X POST http://cocoro.local:8001/memory/archive \
  -H "Authorization: Bearer $COCORO_API_KEY"

# 統計情報の確認
curl http://cocoro.local:8001/memory/stats \
  -H "Authorization: Bearer $COCORO_API_KEY"
```

## 記憶を活用した会話例

記憶システムを最大限活用するプロンプト例：

```
「先月話したアイデアで、まだ実行できていないものを教えて」
→ search_memory("アイデア 未実行") で自動検索

「私の仕事スタイルの特徴を踏まえて、今日のタスクの進め方を提案して」
→ search_memory("仕事スタイル 効率") + list_recent_tasks で統合提案

「去年の今頃は何の話をしていたっけ？」
→ 過去の会話記録から1年前の会話を検索・要約
```
