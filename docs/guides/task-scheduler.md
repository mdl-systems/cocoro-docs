# 🗓️ タスクスケジューラー

Cocoro OS はスケジュール管理とタスク管理を AI が自然言語で受け付けます。
「明日の 14 時に MTG を入れて」と話すだけで登録完了です。

---

## スケジュール管理 vs タスク管理

| | スケジュール（予定）| タスク（ToDo）|
|--|---------------|-------------|
| **特徴** | 日時が決まっているイベント | 期限付き・なしの作業 |
| **例** | 「会議」「アポ」「予約」| 「資料作成」「コードレビュー」|
| **API** | `/schedules` | `/tasks` |
| **Function** | `add_schedule`, `list_schedules` | `create_task`, `list_recent_tasks` |

---

## スケジュール管理

### 予定を追加する

```
「明日の午後3時に鈴木さんとのランチを2時間入れて。場所は渋谷の和食屋」

Cocoro:  [add_schedule] {
           "title": "鈴木さんとのランチ",
           "date": "2026-03-15",
           "time": "15:00",
           "duration_minutes": 120,
           "location": "渋谷の和食屋"
         }

         ✅ 登録しました！
         📅 2026年3月15日（日）15:00〜17:00
         📍 渋谷の和食屋
```

### 繰り返しスケジュール

```
「毎週月曜9時に週次レビューを30分登録して」

Cocoro:  [add_schedule] {
           "title": "週次レビュー",
           "time": "09:00",
           "duration_minutes": 30,
           "recurrence": "weekly",
           "day_of_week": "monday"
         }

         ✅ 毎週月曜 9:00〜9:30「週次レビュー」を登録しました。
         次回: 2026年3月16日（月）
```

### 週間スケジュールを確認

```
「今週の予定を教えて」

Cocoro:  [list_schedules] { "range": "this_week" }

         📅 今週のスケジュール (3/11〜3/17)

         月 3/11:  09:00 週次レビュー（30分）
         火 3/12:  なし
         水 3/13:  14:00 設計レビュー（2時間）
         木 3/14:  なし
         金 3/15:  15:00 鈴木さんとランチ（2時間）
         土・日:   予定なし

         今週は 3件 / 合計 4.5 時間の予定があります。
```

## タスク管理

### タスクを作成する

```
「来週末までにAPI仕様書を書くタスクを高優先で登録して。
  工数見積は4時間くらい」

Cocoro:  [create_task] {
           "title": "API仕様書の作成",
           "priority": "high",
           "deadline": "2026-03-21",
           "estimated_hours": 4,
           "status": "pending"
         }

         ✅ タスクを登録しました。
         📋 API仕様書の作成
         🔴 高優先 | 期限: 3/21（土）| 見積: 4h
```

### タスク一覧を確認

```bash
# API で全タスクを取得
GET /tasks?status=pending&priority=high&limit=10

{
  "tasks": [
    {
      "id": "task_001",
      "title": "API仕様書の作成",
      "priority": "high",
      "deadline": "2026-03-21",
      "status": "pending",
      "progress": 0,
      "estimated_hours": 4
    }
  ],
  "total": 8
}
```

### 進捗を更新する

```
「API仕様書、60%まで終わったよ」

Cocoro:  [update_task] {
           "id": "task_001",
           "progress": 60,
           "status": "in_progress"
         }

         ✅ 進捗を更新しました。
         📋 API仕様書の作成: 60% 完了
         残り: 約 1.6 時間（期限まで 7 日）
```

## リマインダー

### 会話でリマインダーを設定

```
「今日の18時に、API仕様書のリマインダーをかけて」

Cocoro:  ✅ リマインダーを設定しました。
         ⏰ 今日 18:00 — 「API仕様書の作成」をリマインドします。
```

### API でリマインダーを設定

```bash
POST /schedules/reminder
Content-Type: application/json

{
  "task_id": "task_001",
  "remind_at": "2026-03-14T18:00:00+09:00",
  "message": "API仕様書の進捗確認"
}
```

## タスクの優先度システム

| 優先度 | 記号 | 意味 | 表示色 |
|--------|------|------|--------|
| `critical` | 🔴 | 今すぐ対応が必要 | 赤 |
| `high` | 🔴 | 今週中に対応 | 橙〜赤 |
| `normal` | 🟡 | 通常の作業 | 黄 |
| `low` | 🟢 | 余裕があるときに | 緑 |

## 今日やるべきタスクを聞く

```
「今日やるべきことのトップ3を教えて」

Cocoro:  [list_recent_tasks] { "status": "pending", "sort": "priority" }
         [list_schedules] { "date": "today" }

         今日（3/14）やるべきトップ3:

         1. 🔴 API仕様書の作成
            期限まで 7 日、進捗 60%
            → 残り 1.6h。今日中に仕上げられそう

         2. 🟡 コードレビュー（田中さん PR）
            期限: 今日
            → 積み残し注意！

         3. 📅 15:00 設計レビュー（2時間）
            → 14:30 に準備を開始するのがベスト

         今日の空き時間: 午前（約 3h）/ 午後 13:00〜15:00（2h）
```

## スケジュール × タスクの統合管理

Cocoro OS はスケジュール（予定）とタスク（ToDo）を統合的に管理します：

```
「来週の作業計画を立てて。今ある未完了タスクをスケジュールに組み込んで」

Cocoro:
  現在の未完了タスク（高優先）:
  - API仕様書 残り1.6h
  - ユニットテスト実装 残り3h

  来週の空き時間（予定を除く）:
  - 火曜: 09:00〜12:00（3h）
  - 水曜: 09:00〜11:00（2h）
  - 木曜: 終日空き

  提案スケジュール:
  火 3/17 09:00-10:40: API仕様書完成（1.6h）
  火 3/17 11:00-12:00: ユニットテスト開始（1h）
  水 3/18 09:00-11:00: ユニットテスト完成（2h）

  登録してよいですか？ [はい / 修正する]
```

## よく使うコマンド集

| 操作 | 会話での指示例 |
|------|-------------|
| 予定を追加 | 「明日の14時に〇〇を入れて」 |
| 予定確認 | 「今週の予定を教えて」 |
| タスク追加 | 「〇〇タスクを高優先で登録して」 |
| タスク確認 | 「未完了タスクを優先度順で教えて」 |
| 進捗更新 | 「〇〇タスク、50%終わった」 |
| 今日の計画 | 「今日やるべきこと上位3つを教えて」 |
| 来週の計画 | 「来週の作業計画を立てて」 |
| リマインダー | 「18時に〇〇のリマインドをかけて」 |
