# 🧠 メモリシステムの仕組み

---

## 3 層メモリアーキテクチャ

```
                    入力（会話・経験）
                         │
              ┌──────────┼──────────┐
              ▼          ▼          ▼
         ┌─────────┐ ┌──────────┐ ┌─────────────┐
         │短期メモリ│ │長期メモリ │ │ベクトルメモリ│
         │  Redis  │ │PostgreSQL│ │  pgvector   │
         └────┬────┘ └────┬─────┘ └──────┬──────┘
              │           │              │
         セッション中   重要情報を    意味的に近い
           の作業記憶   永続保存     記憶を検索
```

---

## 短期メモリ（Working Memory）

**Redis** に保存。セッション（会話）が終わると消える。

- 現在の会話コンテキスト（直近 20 往復程度）
- 作業中のタスク状態
- 一時的な感情フラクチュエーション

```python
# セッション例
redis.setex(f"context:{session_id}", 3600, json.dumps({
    "history": [...],
    "current_task": "...",
    "emotion_drift": 0.1
}))
```

---

## 長期メモリ（Episodic Memory）

**PostgreSQL** に保存。重要な経験・知識が永続化される。

重要度スコア（0.0〜1.0）が高い記憶ほど長く保持：

| 重要度 | 例 | 保持期間 |
|--------|-----|---------|
| 0.9 以上 | 誕生日・好き嫌い・大切な出来事 | 永続 |
| 0.7〜0.9 | 仕事・趣味・日常の会話 | 数ヶ月〜永続 |
| 0.5〜0.7 | 一般的な会話 | 数週間 |
| 0.5 未満 | 些細な情報 | 自動削除 |

---

## ベクトルメモリ（Semantic Memory）

**pgvector** に保存。テキストを埋め込みベクトルに変換して保存。

```sql
-- 類似した記憶を検索（コサイン類似度）
SELECT content, 1 - (embedding <=> $1) AS similarity
FROM memories
WHERE 1 - (embedding <=> $1) > 0.7
ORDER BY similarity DESC
LIMIT 5;
```

「好きな食べ物は？」と聞くと、
「ラーメンが好き」という過去の会話を自動的に引き出せます。

---

## メモリ統合（Memory Consolidation）

会話セッション終了時に「メモリ統合」が自動実行：

1. 短期メモリから重要情報を抽出
2. 重要度スコアを計算
3. 閾値以上のものを長期メモリに移行
4. テキストをベクトル化して pgvector に保存
5. 短期メモリをクリア

```
毎晩 3:00 AM に「夜間統合」も実行される。
（Archiverモジュールによる自動処理）
```
